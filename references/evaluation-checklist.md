# 批量 Prompt 评测清单

## 小样本集

全量调用前先准备 20-50 条样本：

- 正常高频样本
- 边界样本
- 空值或缺字段样本
- 超长输入样本
- 多标签冲突样本
- 不属于任务范围的样本
- 预期会触发人工复核的样本

## 自动检查

至少检查：

- JSON 可解析率
- schema 通过率
- 必填字段完整率
- 枚举值合法率
- `id` 是否和输入一致
- 是否出现 Markdown、解释性前后缀或代码块
- 输出 token 是否异常偏长

## 人工抽检

分类任务关注：

- 标签定义是否被稳定执行
- 标签优先级是否正确
- `unknown` 是否被滥用或过少使用
- 低置信度样本是否进入 `needs_review`

抽取任务关注：

- 字段是否来自输入原文
- 是否幻觉补全不存在的信息
- 空值策略是否一致
- 证据字段是否能支持结论

改写和摘要任务关注：

- 是否保留关键信息
- 是否引入未给定事实
- 风格和长度是否稳定
- 禁用内容是否被过滤

## 通过标准

建议上线门槛：

- JSON 可解析率 >= 99.5%
- schema 通过率 >= 99%
- 关键字段完整率 >= 99%
- 人工抽检准确率达到业务要求
- 所有 P0 边界样本行为符合预期

如果达不到门槛，优先修 prompt contract 和示例，不要直接扩大样本量。
